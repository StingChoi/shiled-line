# 쉴드광택 자동 발행 시스템 (홈페이지 + 유튜브 + 블로거)

> 목적: 작업 사진·영상을 폴더에 넣어두면, 코워크가 글을 쓰고 → 홈페이지·유튜브·블로거에 한 번에 올린다.
> 발행 방식: **초안으로 올린 뒤 대표님이 확인하고 게시** (자동 공개 아님)

---

## 1. 역할 분담

**대표님이 하는 일 (이것만)**
- 작업한 사진·영상을 정해진 폴더에, 정해진 이름으로 넣어둔다. (아래 2번)

**코워크(자비스)가 하는 일**
- 새 작업 폴더를 확인한다.
- 사진을 고르고 방향보정·크기조절하고, 영상에서 발수 장면을 뽑는다.
- `블로그작성_가이드.md` 톤으로 글을 쓴다 (질문형 소제목 + FAQ + 스키마).
- `blog/` 에 글·사진·영상을 정리하고 `blog/index.html` 에 카드를 넣는다.

**배치파일 `홈페이지+블로거_한방발행.bat` 가 하는 일 (더블클릭 1회)**
1. `[1/4]` 영상을 **유튜브에 업로드**하고 글에 연결 (`upload_videos.py`)
2. `[2/4]` 홈페이지에 **GitHub Push**
3. `[3/4]` 90초 대기 (사진·영상 인터넷 반영 시간)
4. `[4/4]` **구글 블로거에 초안 발행** (`post_html.py`, 유튜브 영상 임베드)

→ 끝나면 블로거에서 **사진·영상 확인 후 '게시'** 를 누르면 공개.

---

## 2. 폴더·파일 이름 규칙 (대표님이 지킬 것)

작업 사진 보관소: `D:\작업사진\광택전후사진_완성\`

그 안에 작업 1건 = 폴더 1개. 폴더 이름은 이렇게:

```
(날짜6자리) (차종) (색) (작업종류)
예) 260508 GLB250 검정 광택 몰딩복원
   260529 E350 레드 광택 G-TOP G-PRO 듀얼코팅
```

폴더 안 사진/영상 이름:

| 종류 | 이름 예시 | 용도 |
|------|-----------|------|
| 작업 전 | `작업전-01.jpg` ~ | Before(잔기스·산화 등) |
| 작업 중 | `광택중-01.jpg`, `코팅중-01.jpg`, `작업중-01.jpg` | 과정 |
| 작업 후 | `작업후-01.jpg` ~ | After |
| 최종 | `최종-01.jpg` ~ | 완성 컷·시공증명서 |
| 확인세차 영상 | `확인세차1.mp4` ~ | 발수 점검 영상 |

> 같은 자리를 전/후로 찍어두면(예: `작업전-13` ↔ `작업후-13`) 비교 사진으로 쓰기 좋습니다.
> 사진 방향이 세로로 찍혀도 코워크가 자동으로 바로잡습니다.

---

## 3. 새 작업 글 만드는 흐름

1. 대표님: 사진·영상을 위 규칙대로 폴더에 넣는다.
2. 코워크에게 "OO차 OO작업 블로그 써줘" 라고 하거나, 정기 점검(아래 5번)으로 코워크가 알아서 초안을 만든다.
3. 코워크가 `blog/` 에 글·사진·영상을 정리한다.
4. 대표님: `홈페이지+블로거_한방발행.bat` 더블클릭.
5. 블로거에서 확인 후 '게시'.

---

## 4. 영상 처리 방식 (중요)

- **홈페이지**: 영상을 직접 올려서 재생한다(`blog/videos/...`). 유튜브 로고 없이 깔끔.
- **블로거**: 자체 영상을 막기 때문에 **유튜브 영상으로 자동 교체**해서 보여준다.
- 그래서 글의 `<video>` 태그에는 로컬 mp4(홈페이지용)와 `data-yt`(유튜브 ID, 블로거용)가 함께 들어간다.
- `upload_videos.py` 가 유튜브 업로드 후 `data-yt` 를 자동으로 채운다. 이미 채워진 영상은 다시 안 올린다.

---

## 5. 정기 자동 점검 (선택, 다음 단계)

- 코워크가 매일 정해진 시간에 `작업사진` 폴더를 확인해서, 아직 글 안 쓴 새 작업이 있으면 초안을 만들고 알려주도록 예약할 수 있다.
- 유튜브 업로드 파이프라인이 한 번 정상 작동하는 걸 확인한 뒤 켜는 걸 권장.
