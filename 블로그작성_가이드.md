# 쉴드광택 블로그 작성 가이드 (코워크 전용)

> 이 폴더(shiled-line)에서 블로그 글을 만들 때는 **반드시 이 가이드를 먼저 읽고** 그대로 적용한다.
> 이 글은 쉴드광택 대표 최한중이 직접 쓰는 톤을 재현하기 위한 것이다.

---

## 0. 가장 중요한 원칙 (이것부터 지킨다)

1. **두괄식**: 첫 두세 줄 안에 "무슨 차를 / 어떤 시공을 / 왜 그렇게 했는지" 핵심이 나와야 한다. AI 검색·네이버 노출에 유리하다.
2. **담백하고 직설적인 기술자 어투**. 홍보성 미사여구는 절대 금지.
3. **한 문장 = 한 줄**. 2~4어절마다 줄을 바꾼다. 빈 줄로 호흡을 끊는다. 긴 단락 금지.
4. **16년 경력 + 직접 제조 + 직접 시공**이라는 사실을 자연스럽게 녹인다. 이게 경쟁사가 흉내 못 내는 핵심이다.

---

## 1. 말투 규칙

- 기본은 **합쇼체**("~합니다", "~했습니다").
- 아쉬움·감탄은 구어체 살짝 허용("참 씁쓸한 현실입니다", "이게 충격적인 결과죠").
- `^^` 는 아주 가끔, `!!` 는 강조할 때만. 이모티콘 최소화.
- **금지 표현**: "최고의", "탁월한", "압도적", "고객만족", "완벽한 서비스" 같은 홍보성 미사여구.
- **금지 표현**: "~인 것 같아요", "~아닐까 싶네요" 같은 얼버무림.
- 타 업체를 원색적으로 비난하지 않는다. 대신 "저가 시공의 위험성"을 교육하는 방식으로 대비시킨다.

### 쉴드광택다운 표현 (이런 결이 좋다)
- "진짜 기술자가 작정하고 덤비면"
- "도장면의 생얼"
- "참 씁쓸한 현실입니다"
- "충격적인 결과"
- "무조건 세게 깎는 게 정답은 아닙니다"

## 2. 글 구조 (5단계)

### ① 오프닝 — 업계 문제 또는 차량 상태로 시작
- 가벼운 날씨 인사("오늘 날씨 좋네요~") 금지.
- 바로 본론: 오늘 입고된 차, 또는 업계에서 흔한 문제 하나를 던진다.
- 두괄식으로 이 글의 결론을 먼저 깐다.

### ② 소제목(하이픈) + 설명
- 소제목은 `- 왜 트리플이 아니라 G-PRO 싱글인가` 처럼 하이픈으로 시작.
- 소제목 아래 2~5줄로 짧게 설명.
- 전문 용어를 쓰면 바로 뒤에 쉬운 비유를 붙인다.
  예) "클리어코트(도장면 맨 위 보호막)"

### ③ 과정 — 전처리를 반드시 강조
- 쉴드광택의 차별점은 **전처리**다. 항상 강조한다.
- 디테일링 세차 → 철분·타르 제거 → 마스킹 → 탈지 순서.
- "눈에 안 보이는 오염까지 잡는 게 퀄리티의 시작입니다" 류의 메시지.
- 중간에 ***기술자의 팁** 을 한두 개 끼워넣으면 좋다.

### ④ 결과 — 솔직하게, 과장 없이
- Before/After 변화를 담담하게 서술.
- 사진이 있으면 사진에서 실제로 보이는 것만 묘사한다(없는 걸 지어내지 않는다).

### ⑤ 마무리(CTA)
- 슬로건으로 닫는다: **"어떤 차를 타냐보다 어떻게 타냐가 중요합니다~!"**
- 연락처 안내:
  - 위치: 부산 남구 유엔로220 1F 쉴드광택
  - 예약·상담: 010-3384-1850
  - 대표: 최한중 (직접 상담 및 책임 시공)

## 5. 결과물 형식 (코워크가 파일 만들 때)

블로그 글은 **HTML 파일**로 만든다. 기존 글 `blog/bmw-x6-gpro-coating.html` 을 템플릿으로 삼는다.

작업 순서:
1. `blog/bmw-x6-gpro-coating.html` 의 구조(헤더·스키마·본문·사진 자리·CTA)를 그대로 복사한다.
2. 본문 내용을 이 가이드에 맞게 새로 쓴다.
3. 사진은 `blog/images/YYYYMMDD-차종/` 폴더에 `01.jpg ~ 05.jpg`, `cert.jpg` 형식으로 둔다.
   - **파일명은 반드시 영문/숫자(ASCII)만**. 한글 파일명 금지.
4. 글 맨 위 `<head>` 안의 BlogPosting 스키마(제목·날짜·설명)도 새 글에 맞게 고친다.
5. 새 글을 `blog/index.html`(목록)에도 카드로 추가한다.

⚠️ 코워크는 파일을 **D:\shiled-line 안에만** 만든다. 인터넷 반영은 사용자가 GitHub Desktop에서 Commit 버튼을 눌러야 일어난다. 코워크가 직접 인터넷에 올리지 않는다.

## 6. 글 만들기 전 체크리스트

- [ ] 두괄식으로 시작했는가
- [ ] 한 문장 한 줄, 빈 줄로 호흡을 끊었는가
- [ ] 홍보성 미사여구("최고의/탁월한/압도적")가 없는가
- [ ] 전처리 과정을 강조했는가
- [ ] 16년 경력·직접 제조를 자연스럽게 녹였는가
- [ ] 제품 스펙을 표 기준으로 정확히 썼는가
- [ ] 슬로건 "어떤 차를 타냐보다 어떻게 타냐가 중요합니다~!"로 닫았는가
- [ ] 연락처(부산 남구 유엔로220 1F / 010-3384-1850 / 최한중)가 있는가
- [ ] 사진 파일명이 전부 영문/숫자(ASCII)인가
